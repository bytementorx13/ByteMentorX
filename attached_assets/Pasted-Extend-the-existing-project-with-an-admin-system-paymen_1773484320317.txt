Extend the existing project with an admin system, payment system, and scheduling system.

The current tech stack is:

* Next.js (frontend + API routes)
* MongoDB (database)
* Razorpay (payments)
* Nodemailer (email)

The website already stores service requests in MongoDB.

Add the following functionality.

---

1. SERVICE TYPES

The platform provides 4 services:

1. Web Development (custom pricing)
2. Consultancy (DSA / Interview Prep)
3. CS Fundamentals
4. Project Guidance & Resume Review

These services have different payment flows.

---

2. PAYMENT FLOW RULES

A. WEB DEVELOPMENT

Web development projects require:

Step 1:
User submits request form.

Step 2:
Admin reviews the request in the admin dashboard.

Step 3:
Admin accepts the request and sets a custom project price.

Step 4:
System sends an email to the user with a Razorpay payment link for:
50% ADVANCE PAYMENT.

Step 5:
After the project is completed, the admin can trigger another payment link for the remaining:
50% FINAL PAYMENT.

MongoDB fields needed:

serviceType
totalPrice
advancePaid (boolean)
finalPaid (boolean)

---

B. CONSULTANCY / CS FUNDAMENTALS / PROJECT GUIDANCE / RESUME REVIEW

For these services the payment must happen BEFORE booking the session.

Flow:

Step 1:
User selects service and submits request.

Step 2:
System immediately generates Razorpay checkout.

Step 3:
User completes payment.

Step 4:
After payment success:

* store payment status in MongoDB
* mark request as "paid"
* send confirmation email

Step 5:
Admin later schedules the session from the admin dashboard.
